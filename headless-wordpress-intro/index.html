Headless WordPress Introduction

Published Feb 13, 2022

I think every web developer know WordPress is, in a nutshell it’s a popular content management system built with php that allows you to manage & manipulate posts, pages, custom post types, categories and more. You can create themes of the web you build with WordPress, it’s quite simple but I don’t want to talk about it now.

One of advantages of WordPress is that it can be a headless CMS, headless means that you don’t need html structure in the CMS or you can say the CMS system is separated with its frontend, in other word you just consume data thru its REST API so you could do CRUD with another frontend stack solutions or integrate it to your another app.

Okay, you got the point, so let’s begin to create a simple blog app with headless WordPress. Make sure you have installed PHP, MySQL and NodeJS on your computer.

Creating Database

First you need to create a MySQL database through terminal or MySQL Command Line Client.

CREATE DATABASE simple_blog

Then you can check the database you just create by this command.

SHOW DATABASES

Installing WordPress If the database has been created, now it’s time to download WordPress at https://wordpress.org/download/, then unzip it into whatever folder you want.

In the folder where the wordpress.zip extracted, open a terminal and give it a command

php -S localhost:5000

Now you can access the wordpress installer on browsers via http://localhost:5000

Alright, the website just successfully run! now, let’s setup some configurations.

Then click “Install WordPress” button.

Now login to its admin panel.

Accessing REST API

Congrats! finally the WordPress app is ready to go, you can start create posts, pages etc.

But, how to consume the data thru its REST API? the answer is simple, you just need to access its API route, for example when you want to retrieve posts data you can go to /wp-json/wp/v2/posts.

Note: before doing that, make sure you set “Post name” structure as your default permalink URL structure. Go to Settings > Permalinks > Choose “Post name” option > Save Changes.

You could do CRUD with the REST API, the documentation is here.
